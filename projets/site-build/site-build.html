But : Manipuler le site de ce cours.
Charger les fichiers présents dans le dépôt git dans des objets javas

Modèliser un site

  1. API modèle
    Un site est composé de cours, et un cours est composé de pages.
    Faire un diagramme de classes contenant les notions de site, cours, pages.
    Identifiez les champs de chaque classe, et fournissez les getters setters.
    Tests unitaires
    + Mécanisme de main() pour pouvoir tester avec println() pendant l'écriture du code.
  2. Import
    Parcourir la hiérarchie de fichiers présents dans le dépôt git, lire chaque fichier, chaque répertoire pour construire des objets Page de l'API modèle ; remplir leurs différents champs (titre, contenu...). => regex
    A définir : règles de correspondance entre les fichiers et les objects de l'API.

    Avant de commencer, faire évoluer le modèle pour intégrer l'import ; diagramme de classe.
    Faire évoluer les package pour organiser ce code ; structure simple et lisible.
  3. Composite
    Ecrire un composite pour représenter un site : le noeud racine est le site, les enfants du site sont les cours, les enfants d'un cours sont des pages.
    Remplissez ce composite avec les objets créés en 2.
    Faire une "opération composite" print(), et afficher le composite.
    Intégrez le nouveau code aux tests unitaires.
  4. Sous-pages et hiérarchie de rubriques
    On ajoute aux pages du site la possibilité d'avoir des enfants de type Page.
    Une page sans enfant est une page normale, un noeud terminal ; une page avec enfants est une rubrique.
    Le composite Site nous donne donc un moyen pour fabriquer une hiérarchie de rubriques à partir des pages.
  5. Visitors
    Ajouter au composite un mécanisme de Visiteur
    Ecrire un visiteur de test qui affiche les titres des pages, des rubriques et des cours.
    Trouver un moyen pour visiter une seule page ; un seul cours ; une rubrique ; une rubrique et ses sous-rubriques.
    Ecrire quelques visiteurs : vérification d'erreur dans les pages, find 'n' replace avec regex dans les pages.
    Un visiteur possible : appeler le code qui génère les sommaires des pages ; java pilote l'exécution d'un programme écrit en go.
  6. Commands
    On a une application qui permet de :
    - charger des données à partir d'un site en html brut,
    - analyser le contenu des pages,
    - produire des informations sur le site ou générer des versions modifier des pages.

    Il faut organiser l'utilisation de l'application.
    Comment la piloter, quelles choses peut-on lui demander ?
    Par exemple
    - affiche la liste des pages qui n'ont pas le tag <meta name="description"
    - regénère la table des matières de la page content/java/lambdas.html
    Etablir la syntaxe du dialogue : comment spécifier une commande, liste des options possibles.
    Envisager les différentes manières d'intéragir entre l'application et ses utilisateurs.
  7. Flux
