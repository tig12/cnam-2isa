Entraînez vous à suivre précisément l'énoncé, comme si vous aviez à faire à un cahier des charges, et que votre code allait subir une code rewiew.
En particulier, veillez à :
  • Respecter les conventions java pour le nomage des classes, variables méthodes

  • Chercher à reproduire exactement l'output demandé

Utilisation des issues github
Visitez https://github.com/tig12/cnam-2isa/issues

Vous pouvez créer une issue pour poser une question, demander de l'aide, donner une information, signaler un bug, suggérer une amélioration etc.
Lire https://github.com/tig12/cnam-2isa/labels

L'important avec les issues, c'est d'avoir un seul sujet précis par issue.
  • Avant de faire une issue, on regarde s'il n'y a pas déjà une issue sur le sujet.
  • Dans une issue, si un sujet en amène un autre, on ouvre une nouvelle issue.
Pour démarrer, allez sur l'issue INSTALL : https://github.com/tig12/cnam-2isa/issues/1 et postez un commentaire pour voir si ça marche.

Si vous rencontrez des problèmes lors de l'installation de git, utilisez l'issue "Install git" : https://github.com/tig12/cnam-2isa/issues/2

Si vous rencontrez des problèmes pour installer java, créez une issue "Install java" (si elle n'existe pas déjà).
Même chose pour l'installation de python.
