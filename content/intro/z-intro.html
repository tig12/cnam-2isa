Paradigmes

On suit l'intro de Paradigmes de programmation - Une introduction, Olivier PORTE, CNRS, Mai 2012 - p 10 à 48.

A retenir :
  • Impératif
    Procédural, objet
    Python : 
    def factorielle1(n):
    if n == 0:
        return 1
    res = 1
    for i in range(1, n+1):
        res *= i
    return res
    Java : 
    public static int factorielle(int n){
    if(n == 0){
        return 1;
    }
    int res = 1;
    for(int i=1; i <= n; i++){
        res *= i; // res = res * i
    }
    return res;
}
  • Déclaratif
    Fonctionnel, logique
    Haskell : 
    factorielle :: (Integral a) => a -> a
factorielle 0 = 1
factorielle n = n * factorielle (n - 1)

    Python : 
    def factorielle2(n):
    if n == 0:
        return 1
    return n * factorielle2(n - 1)
Manière actuelle de traiter les langages au niveau théorique : un langage de programmation est constitué de types de données et de transformations qui mènent d'un type à l'autre. 
// char --> bool
isVoyelle('a') = true
Utilise la théorie des catégories (isomorphisme de Curry - Howard - Lambek).

Lien très fort entre l'informatique et la logique, exprimée par l'isomorphisme de Curry - Howard.
Un langage de programmation peut être vu comme des transformations opérées sur des types de données.
On a :
  • Type de donnée ⇿ chose connue ; axiome ou théorème démontré
  • Application d'une fonction ⇿ démonstration d'un théorème.
